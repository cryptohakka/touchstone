touchstone falsifies the signals your agents trade on — including the ones an LLM produces. Feed it any signal; it runs a five-stage statistical gauntlet and returns one machine-readable verdict, before you let a model act on capital.

two axes, both orthogonal to profit

touchstone falsifies on two separate axes

It never tells you whether a strategy makes money. It asks two independent questions: is the signal real, and is the layer that acts on it actually selective? A thing can pass one and fail the other. Each axis has its own worked example below.

case study · gate_selectivity

The same agent's council blocked 193 trades. Verdict: NOT_SELECTIVE.

A blocking layer can look decisive and still be noise. touchstone's second axis asks whether the council's 193 vetoes beat random selection from the same firing pool. They don't.

gate_selectivity · the same agent's council vs a known-good control
PercepTrade LLM council — 193 real blocks 14.8th percentile vs random selection NOT_SELECTIVE
Oracle gate — hindsight-optimal: blocks the 193 worst by PnL 0th percentile · planted positive control SELECTIVE

The council blocked with apparent conviction, yet its veto was statistically indistinguishable from chance. A gate that is selective — the oracle control — lands at the 0th percentile. The test tells them apart.

is the skeptic just broken?

The harness proves its own detection power

A tool that returns NO_EDGE for everything is useless. So touchstone tests itself: inject a known edge into the dead signal and the verdict must flip.

run it

Clone, point it at a signal, read the verdict

No install step, no dependencies, no network. A reviewer onboards from the README and --help alone.

terminal
$ git clone https://github.com/cryptohakka/touchstone
$ cd touchstone

# start here — falsify a signal: {timestamp, price, signal}
$ node src/index.mjs examples/perceptrade-frz/input.json --map price=btcPrice signal=frZ
→ signal_alpha: NO_EDGE

# then: a deliberately planted edge survives
$ node src/index.mjs examples/synthetic-leak/input.json
→ signal_alpha: SURVIVES

# finally: prove the harness's own power on the dead signal
$ node src/index.mjs examples/perceptrade-frz/input.json --map price=btcPrice signal=frZ --selfcheck
→ selfcheck: PASS  # first SURVIVES at α=0.35

Every real signal tested — my own funding-rate strategy, classic RSI, pure noise — comes back NO_EDGE. The only things that survive are the leaks planted on purpose. That is the tool working, not failing.
