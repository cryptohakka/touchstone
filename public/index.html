what it answers

Two verdicts, both orthogonal to profit

touchstone does not tell you whether a strategy makes money. It tells you whether there is anything real underneath it — a necessary condition, not a sufficient one.

signal_alpha
NO_EDGE/SURVIVES

Does the signal still predict forward returns after the five-stage gauntlet? Most don't. Surviving means the apparent edge is not explained by overlap, autocorrelation, market beta, multiple comparisons, or a few long episodes.

gate_selectivity
NOT_SELECTIVE/SELECTIVE

Does a blocking layer — an LLM council, a rule gate, a hybrid — reject worse-than-random opportunities? Or is its veto, with all its apparent conviction and compute, indistinguishable from a coin flip?

A signal can lose money and still be real; it can make money this month and be noise. Both axes are reported independent of PnL, on purpose.

the signal_alpha gauntlet

Five ways an edge can turn out to be nothing

Each stage strips away one source of false significance. A signal has to survive all five.

01

Overlapping-window t-stats

Establish apparent significance per horizon × side. This is the number that looks impressive and usually isn't.

02

Autocorrelation correction

Overlapping returns are not independent. Effective sample size shrinks, and inflated t-stats deflate with it.

03

Market-adjusted excess

Strip out beta. An edge that was only riding the market's direction has nothing left once the market is removed.

04

Multiple-comparison control

Six horizon×side cells means six chances at a false positive. Benjamini–Hochberg (M=6) raises the bar accordingly.

05

Episode clustering

A signal that fires for 13 hours straight is one event, not 156 samples. Collapsing runs into episodes is where most survivors finally die.

run it

Clone, point it at a signal, read the verdict

No install step, no dependencies, no network. A reviewer onboards from the README and --help alone.

terminal
$ git clone https://github.com/cryptohakka/touchstone
$ cd touchstone

# falsify a signal: {timestamp, price, signal}
$ node src/index.mjs examples/perceptrade-frz/input.json --map signal=frZ
→ signal_alpha: NO_EDGE

# positive control: a deliberately planted edge survives
$ node src/index.mjs examples/synthetic-leak/input.json
→ signal_alpha: SURVIVES

# prove the harness's own power: inject leak into the dead signal
$ node src/index.mjs examples/perceptrade-frz/input.json --map signal=frZ --selfcheck
→ selfcheck: PASS  # first SURVIVES at α=0.35

Every real signal tested — my own funding-rate strategy, classic RSI, pure noise — comes back NO_EDGE. The only things that survive are the leaks planted on purpose. That is the tool working, not failing.

touchstone is a falsification harness, not a strategy. It answers one question honestly — is there anything real here — so an agent never acts on a measurement artifact with full size and full conviction.
